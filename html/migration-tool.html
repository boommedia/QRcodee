The Printed QR Problem — Read First
Dynamic QR codes from QRCodeChimp point to QRCodeChimp's redirect servers (e.g., qrcodechimp.com/r/abc123). That short URL is burned into the physical print. You cannot intercept or take over those redirects — they're on QRCodeChimp's infrastructure.

What this means: Any QR code that has been physically printed (menus, flyers, packaging, signage) will still redirect through QRCodeChimp until it's reprinted. QRcodee migration creates new QR codes with new slugs that the client deploys as they naturally reprint or update digital channels.
🏆 Why This Is a Competitive Moat
No other QR code platform offers a dedicated migration tool from QRCodeChimp. Most businesses dread switching because they think they'll lose all their data and have to manually recreate everything. QRcodee's migration wizard imports their entire library in minutes — names, destinations, folders, everything — and produces a deployment plan showing exactly which codes need reprinting vs. which are digital-only (instant swaps). This single feature removes the #1 objection to switching: "it's too much work."

Deployment Action Types Explained
💻 Digital Swap — Instant
QR code lives in a digital channel only (website, email signature, PDF, social media). Simply replace the image file or update the embed URL. No printing needed. Client can do this in minutes.
Example: website embeds, email, digital menus, apps
🖨 Reprint Required — Planned
QR code exists on physical printed materials (menus, table tents, flyers, packaging, signs). The old QRCodeChimp code still works until the reprint happens — clients can prioritize based on next print run.
Example: printed menus, table tents, product packaging, signage
✓ Inactive — No Action
Paused or expired QR codes. These can be recreated in QRcodee but don't need immediate deployment action. Good for archiving seasonal campaigns or discontinued promotions.
Example: expired promotions, seasonal campaigns, retired menus
